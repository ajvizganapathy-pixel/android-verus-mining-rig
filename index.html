01 — Software

Quick Start
Mining

Get a single Android phone mining Verus Coin in under 20 minutes — no hardware modification needed.

01
Install Termux (F-Droid)
Download F-Droid from f-droid.org, then install Termux from within F-Droid. Do not use Play Store version.
02
Run install.sh
Download install.sh from this repo, place in Termux home, run it. Compiles ccminer automatically.
03
Edit config.json
Open config.json and replace the wallet address with your own from the Verus Mobile app.
04
Start Mining
Run ~/ccminer/start.sh and check vipor.net for your stats.
termux — bash
$# Update packages
$yes | pkg update && pkg upgrade -y
$# Install dependencies
$yes | pkg install libjansson build-essential clang binutils git -y
$# Fix missing header
$cp /data/data/com.termux/files/usr/include/linux/sysctl.h \
   /data/data/com.termux/files/usr/include/sys
$# Clone ccminer
$git clone https://github.com/Darktron/ccminer.git
$cd ccminer && chmod +x build.sh configure.sh autogen.sh start.sh
$# Build with clang
$CXX=clang++ CC=clang ./build.sh
 
✓ ccminer compiled successfully
✓ Stratum connected to sg.vipor.net:5040
✓ CPU T0..T5: Verus Hashing — 452.3 H/s
Auto-start on boot: Add cd ccminer/ && ./start.sh to /data/data/com.termux/files/usr/etc/bash.bashrc — combined with the AutoStart APK, this makes mining begin automatically on every power-on.

02 — Electronics

Hardware
Components

Each tier of 12 phones runs independently with its own power supply, buck converters, and relay control modules.

⚡
XL4015 Buck Converter
INPUT: 8–36V · OUTPUT: 5A continuous
Steps down 12V PSU output to a clean, regulated 5.05V for each group of 4 phones. Set output with blue trim pot before connecting phones.
3× per tier · 9 total
🔋
MUR460 Fast Recovery Diode
4A · 600V · DO-201AD axial
Installed on bare motherboards between Vbus and Vbat pad. The ~0.8V forward drop gives ~4.2V to Vbat, fooling Android into thinking a battery is present.
1× per bare board
🔌
20A Switching PSU
12V DC · 240W · 100–240V AC input
One PSU powers an entire tier of 12 phones. Add a 20A fuse on the V+ output line. Use 12AWG wire minimum for main power runs.
1× per tier · 3 total
⏱️
NE555 Delay Module
12V · Monostable · Adjustable 0.1s–60s
Triggers a 1–2 second pulse to the phone board's PWR_KEY pad when power is first applied. Adjust the blue trim pot for desired delay before power-on.
1× per bare board node
📡
ESP8266 WiFi Relay Module
2.4GHz WiFi · 1-channel · Tasmota
Flashed with Tasmota firmware for Alexa integration. Sits on the 12V input line to the XL4015 — cuts charging power on Alexa's schedule.
4× per tier · 12 total
🌀
Cooling Fans
12V · 80mm or 120mm case fans
Two fans per tier positioned to blow across the row of phones. Phones mining at full load generate significant heat — airflow is essential for stable long-term operation.
2× per tier · 6 total

03 — Motherboard Mod

USB-A
Wiring Guide

Solder a standard USB-A male connector directly to the phone motherboard's test points for power delivery and data handshake.

USB-A Pin Wire Color Board Test Point Voltage Purpose
Pin 1 (VCC)
Red
 Vbus pad 5.0V Main supply input
Pin 4 (GND)
Black
 GND pad 0V Ground reference
Pin 2 (D−)
White
 D− pad — USB data minus
Pin 3 (D+)
Green
 D+ pad (via 4.7kΩ) — Charger handshake
MUR460 Diode Vbus → Vbat ~4.2V Mimics battery presence
NE555 Output PWR_KEY pad ~3.3V pulse Auto power-on trigger
Finding pads: Search your board's model number + "schematic" or "test points" online. Use a multimeter in continuity mode to trace from the USB port pins or battery connector. The reference image (Image 17) in the repo shows the JUMPER point, 4K5 resistor location, and PSU 4V supply points for a typical Qualcomm/MediaTek board.
